## 1. 目的とターゲット（最重要）
- **主目的**: 個人ブランディング / Web上の名刺。
  - 成功の定義 = 訪問者が「この人は◯◯な人」と一言で理解し、強い印象を持ち、他チャネル（X / note / LinkedIn 等）に繋がること。
  - **リード獲得（受注）は主目的ではない。** 「繋がる」がゴール。
- **想定訪問者**: 採用・イベント主催・取材・登壇依頼・人脈づくりで「この人は何者か」を知りたい人。

## 2. ポジショニング / 軸（メッセージの背骨）
- **主軸**: 「AI × ツールを使いこなして、人や現場の"動き方"を変える実践者」＋ AIコミュニティ「AIシドニー会」主催。
- **土台（信頼の証拠）**: 開発できること（iOSアプリ「ウォタべる」App Storeリリース、Web制作）。ただし**主役ではなく裏付け**。
- **判断の根拠**: 実際に人から相談されるのは「AI活用」「Notion/仕組み化」であり、「開発して」ではない。看板を実需側に合わせる。
- **やめること**: 肩書きの渋滞（Web Developer & Designer / AI / Notion / SNS / iOS / 教育… の並列羅列）。軸を一本に通し、他は「その実例」として下にぶら下げる。

## 3. スコープ
### やること（このサイト）
- ヒーローの看板を「Web Developer & Designer」から**実需側（AI×ツール実践者）**に寄せる。
- 売り込み要素のトーンダウン／自己紹介化。
- 信頼の証を「バニティ数字」から「具体実績」へ。
- ダミー問い合わせフォームの撤去と本物チャネルへの置換。
- 死にリンク・重複ページ・未使用データの掃除。

### やらないこと（スコープ外）
- **非公開日記ツール**: 別プロジェクト。Notion据え置き＋Appleショートカット（iPhone/Mac）でNotion APIに直送する入口だけ手軽化する方針が決定済みだが、本サイトとは分離。本書では扱わない。
- 自前ブログ基盤の拡張（note に寄せるため）。

## 4. 技術スタック（現状＝維持）
- **フレームワーク**: Next.js 14 (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS + CSS変数（`globals.css` の `--foreground` 等でテーマ管理）
- **アニメーション**: GSAP + ScrollTrigger（主）、framer-motion（一部の旧ページ）、Three.js（HeroShader）
- **多言語**: next-intl（`ja` / `en`、デフォルト `ja`）。文言は `src/messages/{ja,en}.json`
- **外部データ**: GitHub REST（`src/lib/github.ts`）、note RSS（`src/lib/note.ts`）
- **デプロイ**: Vercel 想定 / GitHub `ShingoMatsushita/my-profile-site`（`main`）

## 5. サイト構成（現状の描画順 = `src/app/[locale]/page.tsx`）
1. **Hero** — 看板・キャッチ・SNS・CTA・アバター（`HomeClient.tsx`）
2. **Marquee** — 技術キーワードの流れる帯
3. **About** — 数字ブロック / bio / skills / interests / Notionカード / Timeline
4. **Portfolio** — GitHubリポジトリをライブ取得して表示（`PortfolioSection.tsx`）
5. **Services** — 「できること」4枚（現状は受注CTA付き）
6. **Blog** — **note RSS をライブ取得**して表示（`BlogSection.tsx`）※既にnote連携済み
7. **Community** — AIシドニー会
8. **Contact** — SNS一覧＋**ダミーフォーム**
9. **Footer**
