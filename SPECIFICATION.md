# プロジェクト仕様書：Personal Profile & Playground Site

## 1. プロジェクト概要
*   **目的**: 自己紹介、ポートフォリオの提示、および技術的なアウトプット（ブログ）。
*   **コンセプト**: 「Web上の名刺」兼「技術的な実験場」。
*   **ターゲット**: 全世界（特定ターゲットなし）。
*   **言語**: 日本語。

## 2. 技術スタック（モダン・学習重視）
現在のWeb開発のトレンドかつ、将来性の高い技術選定とします。

*   **フレームワーク**: **Next.js 14+ (App Router)**
    *   選定理由: Reactフレームワークの標準。SSR/SSGの理解、最新のルーティング機能（App Router）の習得に最適。
*   **言語**: **TypeScript**
    *   選定理由: 型安全性による開発効率向上。実務でもほぼ必須のスキル。
*   **スタイリング**: **Tailwind CSS**
    *   選定理由: ユーティリティファーストなCSSフレームワーク。コンポーネント指向との相性が良く、デザイン構築が高速。
*   **記事管理**: **MDX (Markdown + JSX)**
    *   選定理由: ブログ記事内でReactコンポーネントを動かせるため、「実験の場」として記事自体をインタラクティブにできる。CMSを使わずファイル管理することでGit管理が可能。
*   **デプロイ先**: **Vercel** (推奨)
    *   選定理由: Next.jsの開発元。Git pushで自動デプロイされるCI/CD環境を体験できる。

## 3. サイトマップ（ページ構成）
1.  **Home (/)**
    *   ヒーローセクション（キャッチコピー）
    *   最新のブログ記事へのリンク
    *   簡単な自己紹介
2.  **About (/about)**
    *   詳細な経歴（タイムライン形式など）
    *   スキルセット（アイコンやグラフで可視化）
3.  **Portfolio (/portfolio)**
    *   制作物一覧（サムネイル、説明、リンク、使用技術）
4.  **Blog (/blog)**
    *   技術記事、思考のアウトプット一覧
    *   個別記事ページ (`/blog/[slug]`)
5.  **Contact (/contact)**
    *   お問い合わせフォーム（Google Forms埋め込み、またはAPI Routesを使用した実装）
    *   SNSリンク一覧

## 4. デザイン・機能要件
*   **デザイン**: シンプル、ミニマル。コンテンツ（文章・実績）が際立つデザイン。
*   **レスポンシブ対応**: モバイルファーストで設計。スマホ・PC両方で最適化。
*   **パフォーマンス**: Core Web Vitalsを意識した高速表示（Next.jsの機能を活用）。
*   **拡張性**: 将来的に機能を追加しやすいディレクトリ構成にする。
